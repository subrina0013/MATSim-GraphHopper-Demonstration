About The Project

What it does

This is an experimental evaluation to understand to what degree complex traffic patterns can be approximated using shortest path routing approach.

Goal

The Goal is to understand if massive shortest path computation can be used as a fast approximation of traffic simulations.

What platforms are used

We conduct a traffic simulation using MATSim (an agent-based model), and a shortest path routing using GraphHopper (a routing engine for route planning).

How we do it

We run the models and compare the output traffic, travel time and other metrics



We model the traffic on the roads that the population (sample) produces.

All trips made by an agent or individual during the course of a day are taken into account.

STUDY AREA.

Our study area comprises of eight administrative divisions in DC-Maryland-Virginia (DMV) including Washington D.C., Montgomery County, Prince George's County in Maryland, and Arlington County, Fairfax County, City of Fairfax, City of Fall church, and City of Alexandra in Virginia

HOW TO RUN THIS PROJECT

Step1

Set up the environments

This project runs on IntellijIdea and uses Java and Python. Make sure your system has those.

Step2

Set up the simulation JAR files.

The JAR files for MATSim and GraphHopper are in out.artifacts folder inside GraphHopper and MATSIm.

Update the path for the JAR files (matched with the location where you saved MATSim_DMV and GraphHopper_DMV folder).

In app.py make sure the location of the JAR files are matched with the location of your JAR files.

In matsim_output.py and gh_output.py make sure the file locations match with the saved file locations.

Step3

Run simulations

Update file input locations in gh_output.py and matsim_output.py.

Go to Run Simulations page

Run MATSim, GraphHopper and Generate Outputs

Step4

Visualize the results

Go to Simulation Results page
